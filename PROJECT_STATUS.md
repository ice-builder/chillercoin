# Crypto Research System: Current Capabilities

## Что это сейчас

Локальная исследовательская среда для крипто-фьючерсов, где объединены:

- загрузка и исследование свечей
- обучение локальной модели на OHLCV-данных
- бэктест сигналов
- ручной анализ графика через UI
- локальное хранилище торговых гипотез
- локальный LLM-слой для синтеза идей поверх этих гипотез

Система работает полностью локально в папке проекта.

## Что уже умеет

### 1. Загружать данные по свечам

Поддерживается:

- загрузка своих CSV-файлов со свечами
- генерация демо-датасета
- скачивание реальных свечей Bybit

Для Bybit уже доступны:

- выбор `symbol`
- выбор `interval`
- выбор `start date`
- выбор `end date`
- выбор `start time (UTC)`
- выбор `end time (UTC)`
- сохранение результата в локальный CSV

### 2. Исследовать рынок в локальном UI

В Streamlit-интерфейсе уже есть:

- просмотр свечей на графике
- окно исследования по выбранному участку истории
- просмотр объема
- просмотр признаков модели
- просмотр вероятностей модели, если загружены артефакты обучения
- таблица по свечам, признакам и таргетам
- сводка train/backtest результатов

### 3. Обучать локальную нейросеть на свечах

Текущий ML-пайплайн умеет:

- загружать OHLCV CSV
- строить импульсные признаки
- размечать `short / flat / long`
- обучать небольшую PyTorch-модель
- сохранять артефакты обучения
- использовать их потом в UI и бэктесте

Текущие признаки включают:

- momentum
- short/slow returns
- range/body features
- volume z-score
- volatility
- EMA gap
- breakout/drawdown признаки

### 4. Делать бэктест

Бэктест уже считает:

- сигналы long/short/flat
- комиссии
- slippage
- cooldown между сменами сигналов
- equity curve
- hit rate
- total return
- max drawdown
- sharpe-like metric

### 5. Хранить торговые гипотезы локально

Сейчас есть отдельный локальный vault:

- хранение в SQLite
- отдельный конфиг локального AI
- сохранение названия гипотезы
- сохранение тезиса
- сохранение evidence
- сохранение тегов
- сохранение symbol
- сохранение timeframe
- сохранение status
- сохранение score

Физически это лежит в:

- `.local_ai/config.json`
- `.local_ai/hypotheses.db`

### 6. Использовать локальную LLM как слой осмысления

Локальная модель уже подключена не как замена базы, а как reasoning-слой над гипотезами.

Сейчас доступны:

- просмотр локальных моделей через Ollama
- инициализация локального AI-workspace
- добавление гипотез через CLI
- просмотр гипотез через CLI
- поиск по гипотезам
- synthesis по сохраненным гипотезам

## Какие модели сейчас используются

### Рабочая модель сейчас

Сейчас vault настроен на:

- `llama3.1:latest` как активную reasoning-модель

Это потому, что она уже была установлена локально и сразу заработала.

### Рекомендуемая модель на будущее

Под `Mac mini M4 16 GB` рекомендована связка:

- `qwen3:4b` как основной reasoning-модуль
- `embeddinggemma` как модель для semantic search

Но загрузка `qwen3:4b` сейчас уперлась в очень медленный download через Ollama, поэтому система временно работает на уже установленной `llama3.1:latest`.
